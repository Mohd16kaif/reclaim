GENERAL NOTES
  - This is an Expo-managed React Native app (SDK 54) with heavy native iOS
    integrations: Family Controls (Screen Time), Network Extension (VPN/proxy),
    Device Activity monitoring, Live Activities, and app extensions.
  - No Android-specific features were reviewed beyond what exists in shared JS.
  - Bundle ID: com.reclaim.recovery
  - No custom server infrastructure beyond Supabase (DB + auth + storage +
    Edge Functions) and PostHog (analytics). No Firebase, no Amplitude, no
    Mixpanel, no AppsFlyer, no Adjust, no Branch.

      - Created by: supabase.ts:21-28 — getDeviceId()

        `device_${Date.now()}`
      - Stored in: AsyncStorage key "@reclaim_device_id"
      - This is distinct from the auth UID.
      - Used as: fallback user ID when anonymous auth fails (supabase.ts:39,47,51)
      - NOTE: App.tsx:466 reads AsyncStorage key "deviceId" (no @reclaim prefix)
        which is a DIFFERENT key from the one set in supabase.ts. This may be an
        orphan from an older onboarding flow.

  5.3 PostHog distinct_id
      - analytics.ts:52 — client.identify(deviceId, traits)
      - Uses the device_id passed from App.tsx:477, which reads from
        AsyncStorage key "deviceId" (see note above — may be stale/orphaned).
      - Not a true advertising identifier. This is the app's own generated ID
        or the iOS identifierForVendor. Not IDFA.

  5.4 No advertising identifiers (IDFA)
      - expo-ads, expo-tracking-transparency, or any ad-related library is
        NOT in package.json. No references to ASIdentifierManager or
        advertisingIdentifier found.

  5.5 Apple Identity Token
      - Used at runtime during sign-in (supabase.ts:110,163) but not persisted
        beyond the sign-in flow. Not stored in AsyncStorage or SecureStore.
